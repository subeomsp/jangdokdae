## 결과

- **전체: 369/411 = 89.8%** 성공(본문 추출 성공·최소 길이 통과).
- **investing.com 제외 시: 369/394 = 93.7%** — investing.com(kr 현지화판)은 WAF 403으로 전건 실패라 사실상 수집 불가.

| 매체 | 성공/전체 | 성공률 |
| --- | --- | --- |
| investing.com | 0/17 | 0% (WAF 403) |
| 이데일리 | 38/49 | 78% |
| 서울경제 | 43/48 | 90% |
| 아시아경제 | 61/67 | 91% |
| 뉴스핌 | 19/20 | 95% |

요청량: 윈도우 전체 411건을 동시성 5로 1회 fetch(`follow_redirects=True`).

## 해석

- **전체 fetch 기준 89.8%는 구 설계 대표1건 92%와 사실상 동급** — 전체로 넓혀도 성공률이 무너지지 않음을 확인. `follow_redirects=True`(http→https 301 추적)가 국내 매체 성공률을 지탱한다.
- 실패는 ① investing.com WAF 403(외부 제약, 영어판 아님 → 손실 작음) ② 이데일리 등 일부 페이월/추출 미달. 본문 실패 기사는 제목만으로 임베딩(설계 04 §6 폴백)이라 파이프라인은 정상 동작.

## 한계·주의

- 표본은 한 연속 윈도우(06-16~17). 대형 이벤트일·시간대별 변동은 운영 누적으로 추가 관측 권장.
- 페이월/추출미달의 세부 사유 분해(403 vs 빈 본문 vs 과소추출)는 본 측정에서 미구분 — 필요 시 fetcher 로깅 보강.
