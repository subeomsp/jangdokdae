## 1. 한눈에 보기 (결론)

- **투자 관련 뉴스는 7개 frame으로 충분히 분류된다.** 85건 중 72건이 7개 체계 안에서 적합도 만점(5)으로 들어맞았다(85%).
- **7개가 못 잡는 한 부류가 (표본을 키워도) 일관되게 존재한다 — "비투자성 기업 활동(홍보·사회공헌·ESG·마케팅·교육·부고)".**
  6건(약 **7%**)이 *어느 frame에도 본질적으로 맞지 않음*으로 판정됐고, 5건은 `PLAN`에·1건은 `INCIDENT`에 억지로 흡수돼 있다.
  → 표본 확대로 **비중은 16%→7%로 내려갔지만 부류 자체는 사라지지 않았다**(=`PLAN` 과부하의 실체, 확대 표본에서도 PLAN 34건으로 최다).
- 즉 **"7개를 늘려야 하나?"의 답은 "투자 뉴스 한정 No, 비투자성 PR 부류 때문에 조치 필요 Yes"** 다.
  새 frame 추가보다 **상류 relevance 필터로 비투자성 뉴스를 분석 대상에서 제외**하는 쪽이 큐레이션 취지에 맞다(§5).
- **신규 발견(표본 확대): OPINION 과소검출.** 전문가 칼럼·증권사 리포트·운용사 인터뷰 다수가 OPINION이 아니라
  PLAN/TREND/POLICY로 적재돼 있었다(독립 판정은 OPINION). 7개 안의 일관성 문제이나 빈도가 높다(§3.2·§4).
- 부수 발견(유지): `confidence`는 신뢰 불가(85건 중 72건이 0.9, 명백한 억지 분류도 0.9). 미스핏 판별엔 못 쓴다.

| 지표 | 값 (2026-06-18 재실행) | 초판(32건) |
|------|-----|-----|
| 판정 대상 | **85건** (size≥2, 2개 일자) | 32건 (1개 일자) |
| stored frame = 독립 재판정 일치 | 64/85 (**75%**) | 27/32 (84%) |
| 적합도(fit_score) 분포 | 5점 72건 · 4점 7건 · 1점 6건 | 5점 25 · 4점 2 · 1점 5 |
| 7개 체계 밖(fits_within_seven=false) | **6건 (7%)** — 비투자성 PR/CSR/ESG/교육/부고 | 5건(16%) |
| 7개 안 frame 경계 불일치 | **21건** (OPINION 과소검출·거시 경계) | 5건 |

## 2. 검증 방법

저장된 `confidence`가 과신이라(거의 전건 0.9) 미스핏 판별에 쓸 수 없다. 그래서 **독립 재판정**을 했다:

- 각 이슈를 **원문(대표기사 제목+본문+서브 헤드라인)**으로 다시 구성([NewsAnalyzer._build_issue](../../services/pipeline/news_analyzer.py) 재사용, 본문 재fetch).
- **기존 분류 결과를 가린 채(blind)** 독립 LLM(Gemini)에게 7개 frame 정의와 함께 판정시킴.
- 구조화 출력: `best_frame`(7중 1) · `fit_score`(1~5) · `fits_within_seven`(bool) · `suggested_category`(미스핏 시 필요한 새 분류명) · `reason`.
- 재현: `GOOGLE_CLOUD_PROJECT=<vertex-project> GOOGLE_APPLICATION_CREDENTIALS= uv run python -m scripts.verify_frame_coverage --min-size 2` → 상세 판정은 `/tmp/frame_coverage_verdicts.json`. (적재된 `news_analysis` **전체**를 판정하므로, 표본을 키우려면 먼저 `scripts.run_analysis`로 더 많은 클러스터를 분석해 둔다 — [03 §A](./03-pipeline-stage-evaluation.md).)

판정 기준은 "이 뉴스를 독자가 어떻게 받아들여야 하는가"이며, 7개 어디에도 본질이 안 맞으면(예: 투자 판단과 무관한 홍보·사회공헌) `fits_within_seven=false`로 두고 필요한 새 분류명을 제안하게 했다.

### 3.3 적합도 분포

- **5점(완벽) 72건 · 4점 7건 · 1점(억지) 6건**, 2·3점 없음.
- 즉 분류는 "잘 맞거나(투자 뉴스)" 또는 "전혀 안 맞거나(비투자 잡뉴스)"로 양극화된다. 6건의 1점이 곧 §3.1의 결손 범주다.
- 양극화 패턴은 표본을 32→85건으로 키워도 유지됐다(2·3점 여전히 0). 체계 적합/부적합의 경계가 뚜렷하다는 방증.

## 4. 패턴 분석

1. **`PLAN`이 만능 통(catch-all)으로 과부하.** 확대 표본 85건 중 PLAN이 34건으로 최다인데, 그중 5건은 투자 판단과
   무관한 PR/CSR/ESG/마케팅이 "회사가 뭔가 한다"는 이유로 PLAN에 흡수된 것이고, 추가로 여러 건은 본질이 OPINION인데
   "계획을 알린다"로 읽혀 PLAN이 됐다(§3.2-a). PLAN의 신호가 두 방향으로 희석된다.
2. **결손 범주는 새 *투자 frame*이 아니라 *비투자성 뉴스*다.** 7개 frame은 "투자 판단을 어떻게 할까"의 축이라,
   애초에 투자 판단이 필요 없는 뉴스(홍보·사회공헌·교육·부고)는 이 축에 놓을 수 없다. 분류 체계의 결함이 아니라
   **분석 이전 단계에서 걸러졌어야 할 비관련 뉴스**가 들어온 것에 가깝다. 표본 확대로 비중은 줄었으나(16%→7%) 부류는 잔존.
3. **OPINION 과소검출(신규·빈도 높음).** 전문가 칼럼·리포트·운용사 인터뷰가 PLAN/TREND/POLICY로 갈렸다(§3.2-a).
   OPINION은 09번이 P0로 도입한 전용 frame인데, "리포트가 다룬 *소재*"(신제품=PLAN, 업황=TREND)에 끌려가 정작
   "전문가 의견"이라는 본질을 놓친다. 분류 프롬프트에 "기사 주체가 전문가의 평가·의견이면 소재와 무관하게 OPINION" 강화 필요.
4. **거시지표·환율의 frame 경계가 모호.** 물가·통계·환율이 EARNINGS/PRICE와 TREND/POLICY 사이에서 갈린다(§3.2-b).
   체계 결손은 아니나 분류 프롬프트의 거시 처리 가이드가 약하다.

## 5. 결론 및 제안

**결론: 투자 관련 뉴스에 한해 7개 frame은 충분하다(85건 중 85%가 적합도 만점). 표본을 2개 일자 85건으로
키워도 결론은 유지된다. 단, ① 비투자성 잡뉴스 부류(7%)는 7개로 포착되지 않고 PLAN/INCIDENT에 잘못 흡수되며,
② 7개 안에서는 OPINION 과소검출이 가장 흔한 일관성 문제다.**

권고(우선순위순):
1. **✅ 구현(→[04](./04-classification-improvement-history.md)) relevance 필터** — 비투자성 뉴스(사회공헌·ESG 홍보·마케팅
   이벤트·기관 교육·부고/인사)를 분석 대상에서 제외. *상류 별도 단계 대신* 분류기에 `is_investment_relevant` 게이트를
   두어, false면 콘텐츠 생성·`issue_docent` 적재를 건너뛰는 방식으로 반영(2026-06-18).
2. **✅ 구현(→[04](./04-classification-improvement-history.md)) OPINION 분류 가이드 강화** — 분류 프롬프트에 "기사 주체가
   전문가의 평가·의견·목표가·투자전략이면 소재와 무관하게 OPINION 우선" 규칙 추가(2026-06-18).
3. **(미반영·후순위) 비투자성 보조 카테고리 1개** — 예: scope에 `기타(비투자)` 또는 별개 축 플래그. 현재는 relevance
   게이트로 **제외(skip)** 처리하므로 보조 카테고리는 두지 않음(필요 시 후속).
4. **✅ 구현(→[04](./04-classification-improvement-history.md)) 거시·환율 분류 가이드 보강** — "물가·통계·환율 등 거시지표는
   TREND/POLICY 우선" 규칙을 분류 프롬프트에 추가(2026-06-18).

후속: 본 재실행은 2개 run_date(2026-06-15·06-17) size≥2 클러스터 85건 기준이다. 단일 클러스터로 묶이지 않은
size=1 기사(다수)와 추가 일자를 포함하면 비투자성 비중·OPINION 과소검출률이 더 달라질 수 있어 정기 모니터링을 권장한다.
