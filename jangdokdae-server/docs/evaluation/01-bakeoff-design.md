## 1. 목표·성공 기준

백지 상태에서 **임베딩 모델 · 클러스터링 알고리즘 · 입력 구성** 세 가지를 새 골드셋의 **쌍별 F1**(주지표, 보조 ARI·NMI)로 선정한다. 이전 평가의 결론은 가정으로 끌고 오지 않고 데이터로 다시 판정한다.

성공 기준:

- 재현 가능한 bake-off 하니스를 커밋한다(시드 고정·설정 파일화·벡터 캐시).
- 골드셋(연속기간 전수·LLM 제안+사람 검수·태그 포함)을 구축·커밋한다.
- 게이팅으로 **(모델, 알고리즘, 입력 구성) 1셋을 확정**하고 리포트를 남긴다.
- 결론이 갈리지 않을 때의 **베이스라인 폴백**을 정의한다.

## 2. 골드셋 구축

| 항목 | 결정 |
| --- | --- |
| 출처 | **기존 DB 누적분의 연속 윈도우**(06-16~06-17, kept 411건)에서 추출. 신규 수집 대기 0 — 마감 제약 때문에 "신규 전수" 대신 채택. 윈도우 내 전수를 받아 클러스터 경계 왜곡 최소화 |
| 규모 | kept **411건**(LLM 1차 클러스터링이 전수를 처리, 사람은 스팟 검수만 하므로 라벨링 부담은 클러스터·모호 케이스 수에 비례) |
| 본문 캐시 | 입력 구성이 비교축이므로 골드셋 항목은 `trafilatura`로 본문 fetch해 **골드셋 파일에 캐시**한다(실측 본문 ok 369/411=90%). 운영 "본문 미저장" 원칙의 *명시적 예외* — 오프라인 실험 픽스처일 뿐 운영 DB엔 저장하지 않음 |
| 라벨링 | **LLM 1차 클러스터 제안 → 사람 스팟 검수**(모호 케이스 집중). LLM은 **텍스트만** 보고 묶으며 평가 대상 임베딩을 쓰지 않는다(순환 편향 차단). 단일 사건 기준 |
| 태그 | 난이도(언어는 전건 ko — 교차언어 미적용) |
| 산출물 | `scripts/data/goldset_2026-06-16.json`(재현용 커밋) |

## 3. 하니스 구조

신설 패키지 `evaluation/`(문서 폴더 `docs/evaluation/`와 짝). 단일 책임 모듈로 분리하고, 임베딩·클러스터링은 운영 코드를 재사용한다.

| 모듈 | 책임 |
| --- | --- |
| `goldset.py` | 골드셋 로더 + gold 레이블 추출(미라벨 거부) |
| `embeddings.py` | 모델별 제목/제목+본문 가중평균 임베딩(α=0.3, 각 L2 정규화 후 결합) + **디스크 벡터 캐시**. 임베딩 백엔드는 기존 `services/embedder.embedding_client.embed_with`(gemini/HF 분기) 재사용 |
| `metrics.py` | 쌍별 F1(주) + ARI·NMI(보조), `sklearn` 기반 |
| `run.py` | Gate 1 오케스트레이션(모델×입력 → 임베딩 → 클러스터링 → 지표) + md 리포트 |
| (재사용) | 클러스터링은 기존 `services/embedder.cluster`의 `cluster_news`(HDBSCAN precomputed cosine)·`promote_singletons` 사용. 그래프 알고리즘은 Gate 2에서 추가 |

골드셋 추출·라벨링은 데이터 준비라 `scripts/`에 둔다: `build_goldset.py`(DB 윈도우→본문 fetch 캐시), `label_goldset.py`(Gemini 1차 클러스터 제안→gold 레이블 + 검수본 md).

## 4. 게이팅 절차

매트릭스 폭발(모델×입력×알고리즘×파라미터)을 직렬 게이트로 축소한다. **마감(월) 내 Gate 1을 우선 완료**(모델+입력 확정)하고, 시간 여유 시 Gate 2를 이어간다. 파라미터 스윕(Gate 3)은 §8.2로 분리한다.

- **Gate 1 — 모델 컷 + 입력 구성 결정** *(월요일 필수)*: 후보 4모델 × {제목 단독, 제목+본문} 을 HDBSCAN 기본(라이브러리 디폴트) 파라미터로 **한국어 클러스터링 쌍별 F1** 측정 → 하위 모델 탈락 + **이긴 입력 구성 고정**. (교차언어 게이트는 영어 데이터 부재로 제거 — 위 데이터 실측 정정 참조.)
- **Gate 2 — 알고리즘 비교** *(완료 2026-06-21)*: ko-sroberta·gemini × {HDBSCAN, 그래프} 쌍별 F1 → **HDBSCAN 채택**(ko-sroberta 0.610 vs 그래프 best 0.585, gemini도 동일). 구현 `evaluation/clusterers.py`·`gate2.py`, 결과 [02-gate2-result.md 후속 03](./03-gate2-result.md).
- **폴백**: 게이트가 ΔF1 미미(예: ≤0.02)로 갈리지 않으면 운영성·비용으로 결정, 그래도 불명확하면 `gemini-embedding-001 + HDBSCAN`.

후보 모델: **bge-m3 · multilingual-e5-large · gemini-embedding-001 · ko-sroberta(baseline)** (다국어 한·영 필수).

## 5. 에러 처리·재현성

시드 고정 · 모델별 벡터 캐시로 재실행 멱등 · 임베딩 API 일시 실패는 `with_retry` · 리포트에 실행 환경·모델 버전·파라미터를 함께 기록해 결과를 추적 가능하게 한다.

## 6. 테스트

- 지표 함수(쌍별 F1·ARI)를 **소형 합성 라벨**로 단위 테스트.
- 골드셋 스키마 검증 테스트.
- 소형 골드셋으로 하니스 **스모크 1회 완주**(로더→임베딩→클러스터링→지표→리포트).

## 7. 스코프 경계

- **포함**: 골드셋 구축, 하니스 모듈, Gate 1 실행(모델+입력 확정·월요일 필수), 선정 결과 리포트. Gate 2(알고리즘)는 시간 여유 시.
- **제외(후속 spec)**: 파라미터 스윕(§8.2) · 본문 fetch 실측·블롭 재현(§8.3) · reprint·TOP_N 도입 판단(§8.4) · 선정값의 운영 `app/config.py`·`.env` 반영(§5 재구현).
