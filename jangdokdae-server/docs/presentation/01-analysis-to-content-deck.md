# 장표 구성 — 뉴스 분석 → 콘텐츠 생성 파이프라인

> 발표용 슬라이드 구성안 (~11장, 기술+가치 균형). 슬라이드별 **제목 · 핵심 메시지 · 본문 불릿 · 시각자료 · 발표자 노트**.
> **핵심 서사(슬라이드 2~6)**: 페인포인트 → (그래서) 콘텐츠 틀을 **scope×frame**으로 재정의 → **각 frame이 어떤 페인을 푸는지** → 그 틀의 산물 **4-head**.
> 근거 설계: docs/design/06·08·10·12·13·15. 수치·예시는 2026-06-22 운영 데이터 실측.

## Slide 2 — 페인포인트: 고민에서 출발

**핵심 메시지**: 사용자가 말하는 고민과 **진짜 원인**은 다르다.

- 타깃 김주린(28세·투자 8개월): 헤드라인은 읽지만 **공시·리포트는 못 읽는다**
- 6대 고민:

  | ID | 고민 | 사용자가 말하나? |
  |----|------|------------------|
  | **P1** | **뉴스·공시·리포트를 못 읽음(독해)** | ✕ **자기도 모름 — 진짜 원인** |
  | P2 | 또 잃을까 무섭다 | ○ 1위(35.9%) |
  | P3 | 어디서 시작할지 모름 | ○ 2위(28.2%) |
  | P4 | 검색하면 광고·리딩방뿐(불신) | △ |
  | P5 | "그것도 몰라?" 위축 | △ |
  | P6 | 분석할 시간이 없다 | ○ 3위 |

- **★ 통찰**: 사용자는 **손실(P2)·시작점(P3)** 언어로 말 걸지만, 진짜 원인은 **독해(P1)** — 본인도 모른다(설문 최하위 2.6%).
- 전략 한 줄: **"손실·시작점 언어로 말 걸고, 그 안에서 독해를 떠먹인다."**

## Slide 3 — ★ 핵심 전환: 페인이 콘텐츠 '틀'을 정했다

**핵심 메시지**: "독해를 떠먹이려면" → 뉴스를 **'어떻게 읽어야 하는지'로 나눠야** 한다.

```
독해 불능(P1)을 떠먹인다
      │
      ▼
뉴스를 '읽는 법'으로 나눈다  ──►  두 가지 질문
      │
      ├─  누구 얘기인가?      →  scope (3)   ← 뉴스의 고정 성질
      └─  어떻게 읽어야 하나?  →  frame (7)   ← 읽는 눈 = "해설 구조를 정한다"
      │
      ▼
콘텐츠 생성의 틀  =  scope × frame
```
- 막연한 '뉴스 유형 나열'이 아니라, **페인에서 유도된 재정의**. frame이 *어떻게 설명할지*까지 결정한다.
- 보조 통찰: **국내/해외는 분류 축이 아니다** — 읽는 법·개수는 같고 **제목 문구만 바뀌는 flag**.
- 노트: "이 한 번의 재정의가 이후 모든 콘텐츠 구조를 결정했다"가 발표 클라이맥스.

## Slide 4 — scope × frame: 틀의 정의

**핵심 메시지**: 두 축은 독립 — 어떤 조합도 정상.

- **scope(주인공, 3)**: 회사 · 업종·테마 · 시장 전체
- **frame(읽는 법, 7)**:

  | code | 사용자 노출 | 무슨 뉴스 |
  |------|------------|-----------|
  | EARNINGS | 실적이 나왔어요 | 매출·이익 성적표 |
  | INCIDENT | 악재가 생겼어요 | 사고·소송·제재 |
  | PLAN | 새 계획을 발표했어요 | 신제품·투자·수주 |
  | POLICY | 제도가 바뀌어요 | 정책·규제·금리 |
  | TREND | 업황이 달라져요 | 산업 흐름·수급 |
  | OPINION | 전문가가 평가했어요 | 리포트·목표주가 |
  | PRICE | 주가만 움직였어요 | 이유 불명 급등락 |

- 규칙: **원인 우선** — 주가 움직임은 늘 *결과*. 원인(실적·사건·정책)을 먼저 찾아 frame을 정하고, 뚜렷한 원인이 없을 때만 PRICE.
- 보조 축: origin(flag) · direction(상승/하락/중립) · 태그(기업·섹터·용어) · 투자관련성 · confidence

## Slide 5 — ★ 각 frame이 푸는 고민 (페인 루프를 닫는다)

**핵심 메시지**: frame은 그냥 분류가 아니다 — **frame마다 특정 페인을 정조준해 푼다.**

| frame (읽는 법) | 푸는 고민 | 어떻게 푸나 |
|-----------------|-----------|-------------|
| 실적이 나왔어요 (EARNINGS) | **P1 독해** | "9.2조"가 큰지 모르는 이에게 **'예상 대비'**로 판단 기준을 떠먹임 |
| 악재가 생겼어요 (INCIDENT) | **P2 손실공포** | 막연한 공포 대신 **"얼마나·얼마 동안"** 걱정할 일인지 크기를 |
| 새 계획을 발표했어요 (PLAN) | **P4 신뢰** | 보도자료 그대로 안 믿게 **"진짜 돈이 되려면 뭐가 더 필요한지"** 짚음 |
| 제도가 바뀌어요 (POLICY) | **P3 시작점·P6 시간** | 남 얘기 같은 큰 뉴스를 **"그래서 내 돈엔 어떻게 오나"**로 연결 |
| 업황이 달라져요 (TREND) | **P1 독해·P2 손실** | 추상적 "업황 회복"을 **내 종목까지 닿는 그림**으로 |
| 전문가가 평가했어요 (OPINION) | **P4 신뢰·P1 독해** | 목표가를 그대로 안 믿게 **괴리율·전제 가정**을 함께 보여줌 |
| 주가만 움직였어요 (PRICE) | **P2 따라 사기** | "이유 없이 급등"에 휩쓸려 **추격매수 안 하게** 이유를 의심 |

- 메시지: 슬라이드 2의 고민(P1~P6)이 frame별로 **분담·해결**된다 → 페인에서 시작한 이야기가 여기서 닫힌다.
- 시각: 좌(P1~P6) ↔ 우(7 frame) 매핑 라인. P1(독해)·P2(손실)는 여러 frame이 공통으로 푸는 핵심 고민.

## Slide 6 — frame → 4-head: 틀의 산물

**핵심 메시지**: **frame이 정해지면, 그 frame이 답해야 할 head 4칸이 정해진다.** (해설 구조 = frame의 함수)

- 예) EARNINGS 4-head: **이번 실적, 핵심 숫자만 / 시장은 이 정도를 기대했어요 / 이 흐름, 이어질까요? / 앞으로 뭘 보면 될까요**
- frame마다 4칸 질문이 다르다 → "무엇을 짚어줄지"가 frame에서 자동으로 따라온다 (= 그 frame의 페인 해결 방식)
- 함께 생성: **제목**(LLM) · 첫 줄 hook(pain·neutral) · 연결 모듈(섹터별 영향·종목 후보) · 근거/용어 span
- 톤 3원칙: ① 쉬운 단어·어른스런 사고 ② 확신 대신 **구분** ③ **설명(사실)·판단(해석) 분리** + 금지 표현·"결론만 X, 인과 1단계 강제"
- 시각: 실제 콘텐츠 카드 1장 목업(Slide 11 예시 재활용)

## Slide 7 — 파이프라인 한눈에

**핵심 메시지**: 위 틀을 분류→보강→생성→적재의 단일 흐름으로 자동화.

```
news_cluster(상위 이슈)
   → ① 분류(scope×frame) → ② 추가자료 보강 → ③ 4-head 콘텐츠 생성 → ④ 적재 → (⑤ 노출 큐레이션)
```
- LangGraph 단일 에이전트 · Airflow `analyze` Task가 일 1회 실행(주말 포함)
- 시각: 좌→우 플로우 다이어그램

## Slide 8 — 아키텍처: 2-call + 구조화 출력

**핵심 메시지**: 분류·생성을 2번의 LLM 호출로 나누고, 출력은 스키마로 강제.

- **호출 A 분류**(temp 0) → **enrich**(OPINION 현재가 등 추가자료) → **호출 B 생성**(temp 0.3)
- 각 호출 `with_structured_output(Pydantic)` → **JSON 파싱 오류 제거**
- 프롬프트는 코드 밖 `prompts/*.yaml`(frame_head_specs 등) — 비개발자도 수정
- 모델: **Vertex AI Gemini** (LangChain / LangGraph)
- 시각: `classify → enrich → generate` 노드 그래프

## Slide 9 — 데이터 모델 + 관심사 태깅

**핵심 메시지**: 분류는 `news_analysis`, 콘텐츠는 `issue_docent` — 태깅으로 개인화 연결.

- `news_analysis`: scope·frame·origin·direction·confidence·태그(해소 id)·needs_review
- `issue_docent`: 4-head·hook·연결·span + **제목** + **market / sector / company id 태깅**
- 태깅 → 온보딩 관심사(시장·섹터·종목) 기반 **개인화 피드**. market = 종목 거래소(KOSPI/KOSDAQ…), 해외·종목없음은 **GLOBAL**
- 시각: 두 테이블 핵심 컬럼 + (관심사 ↔ 태그) 매칭 화살표

## Slide 10 — 믿을 수 있는 콘텐츠: 품질 게이트

**핵심 메시지**: 발행 가치 없는 콘텐츠는 만들지 않거나 검수 큐로 격리.

- relevance 필터(비투자성 skip) · 본문 부족 사전 차단 · OPINION 종목 가드 · **honest-blank 게이트(무내용 차단)** · 저신뢰 `needs_review` 격리
- term_spans **본문 정합**(본문에 없는 용어 하이라이트 방지)
- 시각: 통과 → 발행후보 / 격리 → 검수 분기도

## Slide 11 — 실증 · 결과 · 향후

**핵심 메시지**: 실제로 돈다 — 2026-06-22 운영 데이터.

**Worked example (cluster 1185, 회사 / OPINION / 국내 / 상승, conf 0.9)**
- 제목(LLM): **"삼성전기, 기판 업황 긍정적 평가에 목표주가 상향"**
- 태깅: market=KOSPI · sector=IT · company=삼성전기 / hook: *"이 목표가, 그대로 믿어도 될까요?"*
- 4-head: **목표가가 나왔어요**(KB증권 220만→300만, +36%) / **지금 가격과 얼마나 다른가요?**(현재가 238.3만 대비 +25.8% ← enrich) / **이 숫자, 믿어도 될까요?**(업황 재평가·반사이익 *가정*) / **앞으로 뭘 보면 될까요**
- → OPINION의 페인(P4 신뢰·P1 독해)을 괴리율·가정으로 정확히 푸는 실제 사례
