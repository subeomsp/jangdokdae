# PPT 기본 정보 브리프 — 분석 → 콘텐츠 생성 ↔ 페인포인트 해결

> PPT 제작용 *원천 정보*(source of truth). 슬라이드 레이아웃은 `01-analysis-to-content-deck.md` 참조.
> 주안점: **분석부터 콘텐츠 생성까지의 흐름이 각 페인포인트와 어떻게 맞닿고 어떻게 해결하는가.**
> 근거: docs/design/08·10·13·15 + 2026-06-22 운영 실측.

---

## 1. 한 줄 핵심

> **뉴스를 '읽는 법(scope×frame)'으로 나눠, 주린이가 못 하는 '독해(P1)'를 콘텐츠로 떠먹인다.**
> — 클러스터링된 뉴스 → 분류 → 추가자료 보강 → 4단 해설 생성 → 적재까지 자동화.

- 정체성: **종목 추천이 아니라 '뉴스 읽는 법'을 가르친다.** (설명=사실 단정 / 판단=해석 헤지 분리)

## 2. 타깃 + 6 페인포인트

타깃 = **김주린**(28세·투자 8개월·헤드라인은 읽지만 공시·리포트는 못 읽는 직장인).

| ID | 고민 | 종류 | 입으로 말하나 |
|----|------|------|----------------|
| **P1** | **뉴스·공시·리포트를 못 읽음 (독해)** | 기능 | ✕ 자기도 모름 (설문 2.6%, 최하위) — **진짜 원인** |
| **P2** | 또 잃을까 무섭다 / 이미 잃음 | 감정 | ○ 1위(35.9%) — 겉으로 호소 |
| **P3** | 어디서부터 시작할지 모름 | 기능 | ○ 2위(28.2%) — 겉으로 호소 |
| **P4** | 검색하면 광고·리딩방뿐, 못 믿음 (불신) | 기능·감정 | △ |
| **P5** | "그것도 몰라?" 위축 | 감정 | △ 잠재 |
| **P6** | 분석할 시간이 없다 | 시간 | ○ 3위(17.9%) |

**★ 통찰**: 사용자는 **손실(P2)·시작점(P3)** 언어로 말 걸지만, 진짜 원인은 **독해(P1) — 본인도 모름.**
→ 전략: **"손실·시작점 언어로 말 걸고, 그 안에서 독해를 떠먹인다."**

## 3. 핵심 전환 서사 (이 발표의 클라이맥스)

```
독해 불능(P1)을 떠먹인다
  → 뉴스를 '어떻게 읽나'로 나눠야 한다
  → 두 질문:  누구 얘기?(scope 3)  ×  어떻게 읽나?(frame 7)
  → 콘텐츠 생성의 틀 = scope × frame   (페인에서 유도된 재정의)
  → frame이 '해설 구조(4-head)'까지 결정 → 콘텐츠가 따라 나온다
```
- **국내/해외는 분류 축이 아니다** — 읽는 법·개수는 같고 제목 문구만 바뀌는 **flag**.
- scope·frame은 **독립** — 어떤 조합도 정상. **원인 우선**: 주가 움직임은 결과(원인 frame을 먼저, 없을 때만 PRICE).

## 4. 파이프라인 4단계 (분석 → 생성) + 페인 연결

| 단계 | 하는 일 | 산출 | 페인 연결 |
|------|--------|------|-----------|
| **① 분류** | scope×frame + origin·direction·태그(기업/섹터/용어)·투자관련성·confidence | `news_analysis` | 어떤 '읽는 법'으로 풀지 결정 → 페인별 해설 틀 선택 |
| **② 추가자료 보강(enrich)** | frame별로 비는 해설 단에 자료 부착(재무/공시 RAG/주가/거시) | enrichment 컨텍스트 | *없으면 못 푸는 페인*을 메움 (§5, 아래 5번 표) |
| **③ 4단 해설 생성** | frame이 질문 4개 고정 → 4-head 작성 + 제목·hook(pain/neutral)·연결모듈·근거/용어 span | `ContentDraft` | 독해(P1)를 떠먹이고, frame별 핵심 페인을 정조준 |
| **④ 적재** | 콘텐츠 + **market/sector/company 태깅** | `issue_docent` | 태깅 → 관심사 개인화(내 종목·섹터·시장) = "남 얘기 아님"(P3·P6) |

- 아키텍처: **2-call**(분류 temp0 → enrich → 생성 temp0.3), 각 `with_structured_output(Pydantic)`. 프롬프트는 `prompts/*.yaml`. 모델 Vertex AI Gemini. LangGraph 단일 에이전트(Airflow 일 1회).

## 5. ★ frame ↔ 페인 마스터 표 (맞닿음 + 해결법)

> 이 브리프의 핵심. "각 frame이 어떤 페인을, 무엇으로, 어떻게 푸는가"를 한 행에.

| frame (읽는 법) | 무슨 뉴스 | 4-head가 묻는 핵심 | 보강 자료(없으면?) | 푸는 페인 | 어떻게 푸나 |
|---|---|---|---|---|---|
| **실적이 나왔어요** (EARNINGS) | 매출·이익 성적표 | 핵심 숫자 / 예상 대비 / 이어질까 / 볼 것 | 재무제표+사업보고서 (없으면 숫자가 큰지 작은지 판단 불가) | **P1·P4** | "9.2조"에 **'예상 대비'** 판단 기준을 떠먹임 |
| **악재가 생겼어요** (INCIDENT) | 사고·소송·제재 | 무슨 일 / 얼마나 영향 / 일시적인가 / 볼 것 | 공시 원문+주가 (없으면 막연한 공포만 커짐) | **P2·P4** | 막연한 공포 대신 **"얼마나·얼마 동안"** 크기를 |
| **새 계획을 발표했어요** (PLAN) | 신제품·투자·수주 | 뭘 하겠다 / 매출 되나 / 남은 숙제 / 볼 것 | 공시 원문+재무 (없으면 보도자료를 그대로 믿음) | **P4** | **"진짜 돈이 되려면 뭐가 더 필요한지"** 짚음 |
| **제도가 바뀌어요** (POLICY) | 정책·규제·금리 | 뭐가 바뀌나 / 누가 이득·부담 / 확정인가 / 볼 것 | 거시지표+업종 매핑 (없으면 남 얘기로 흘림) | **P3·P6** | 큰 뉴스를 **"그래서 내 돈엔 어떻게 오나"**로 연결 |
| **업황이 달라져요** (TREND) | 산업 흐름·수급 | 무슨 변화 / 누가 이득·부담 / 이어질까 / 볼 것 | 동종 기업 보고서+거시 (없으면 추상적 흐름만) | **P1·P2** | 추상적 "업황 회복"을 **내 종목까지 닿는 그림**으로 |
| **전문가가 평가했어요** (OPINION) | 리포트·목표주가 | 목표가 / 현재가 대비 괴리 / 믿어도 되나 / 볼 것 | **주가(현재가→괴리율)**+재무 (없으면 목표가가 공중에 뜸) | **P4·P1** | 목표가를 그대로 안 믿게 **괴리율·전제 가정**을 함께 |
| **주가만 움직였어요** (PRICE) | 이유 불명 급등락 | 주가 움직임 / 진짜 이유? / 변화 있었나 / 볼 것 | 공시·거시 교차확인+주가 (안 뒤지면 추격매수 부추김) | **P2** | "이유 없이 급등"에 **추격매수 안 하게** 이유를 의심 |

→ 슬라이드 2의 고민(P1~P6)이 frame별로 **분담·해결**됨. 공통 핵심 = **P1(독해)·P2(손실)**.

## 6. 믿을 수 있는 콘텐츠 — 품질 게이트

발행 가치 없는 콘텐츠는 **만들지 않거나 검수 큐로 격리** (불신 P4·잘못된 학습 방지):
- relevance 필터(비투자성 skip) · 본문 부족 사전 차단 · OPINION 종목 가드 · **honest-blank 게이트**(무내용 차단) · 저신뢰 `needs_review`
- **term_spans 본문 정합**(본문에 없는 용어 하이라이트 방지)

## 7. 실증 (2026-06-22 운영 데이터)

**Worked example — cluster 1185 (회사/OPINION/국내/상승, conf 0.9)**
- 제목(LLM): "삼성전기, 기판 업황 긍정적 평가에 목표주가 상향" / 태깅: KOSPI·IT·삼성전기
- hook: *"이 목표가, 그대로 믿어도 될까요?"*
- 4-head: 목표가(KB증권 220만→300만, +36%) / **현재가 238.3만 대비 +25.8%**(← enrich 괴리율) / 업황 재평가·반사이익 *가정* / 볼 것
- → **OPINION의 페인(P4 신뢰·P1 독해)을 괴리율·가정으로 정확히 해결**한 실제 사례

**실측**: 분석 58건 · 발행후보 51건 · needs_review 격리 5건 · 비투자성 3건
- frame 분포: PLAN 16 · PRICE 11 · POLICY 8 · OPINION 8 · TREND 7 · INCIDENT 6 · EARNINGS 2
- market 태깅: KOSPI 46 · KOSDAQ 21 · GLOBAL 17 · KOSPI+KOSDAQ 13 · 빈값 32

## 8. 빠른 참조 (부록값)

- **scope(3)**: 회사 · 업종·테마 · 시장 전체
- **frame(7)**: EARNINGS 실적이 나왔어요 / INCIDENT 악재가 생겼어요 / PLAN 새 계획을 발표했어요 / POLICY 제도가 바뀌어요 / TREND 업황이 달라져요 / OPINION 전문가가 평가했어요 / PRICE 주가만 움직였어요
- **페인(6)**: P1 독해 · P2 손실 · P3 시작점 · P4 신뢰 · P5 위축 · P6 시간
- **scope별 우선 보강자료**: 회사→사업보고서(RAG) / 업종·테마→동종 기업 묶음+대표 주가 / 시장 전체→거시지표
- **콘텐츠 10종 ↔ 페인**(설계 08 §6, 본 발표 범위 밖이나 참고): 도슨트(P1)·용어카드(P1)·전문가시각(P1·P4)·리스크해설(P2)·출처배지(P4)·익명 AI(P5)·아침브리핑(P6)·입문트랙(P3)·퀴즈(학습)·커리큘럼(P3)
- 데이터: `news_analysis`(분류) · `issue_docent`(콘텐츠+태깅) · 스키마 `ClassificationResult`/`ContentDraft`
