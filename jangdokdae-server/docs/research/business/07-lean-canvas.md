> 프레임 참고: Ash Maurya, *Running Lean*. **BMC가 사업 구조를 그린다면, 린 캔버스는 스타트업의 리스크(문제·솔루션·지표)에 초점**을 둔다.
> ⚠️ 모든 항목은 **데스크 리서치 + N=56 최종 설문 실측 기반의 가설**이다. 검증 수치엔 출처, 가정·해석엔 **[가정]·[추론]**.

## 0. BMC와의 관계 (무엇이 다른가)

린 캔버스는 [`06-business-model-canvas.md`](06-business-model-canvas.md)의 9블록 중 **4개를 스타트업 리스크 관점으로 교체**한다. 나머지 5개(고객군·UVP·채널·수익원·비용)는 BMC를 그대로 잇는다.

| BMC 블록 | → 린 캔버스 블록 | 왜 바꾸나 |
|---|---|---|
| 핵심 파트너 | **문제(Problem)** | 초기엔 파트너보다 "진짜 문제"가 리스크 |
| 핵심 활동 | **솔루션(Solution)** | 문제에 대응하는 최소 기능 |
| 핵심 자원 | **핵심 지표(Key Metrics)** | 무엇을 측정해 학습할지 |
| 고객 관계 | **불공정 우위(Unfair Advantage)** | 모방 불가한 해자 |

→ 이 문서는 **교체된 4블록(문제·솔루션·지표·불공정우위) + 린 특유 항목(얼리어답터·하이레벨 콘셉트)**에 집중하고, 동일 5블록은 BMC를 참조한다.

## 2. 문제 (Problem) — 린 특유 ★

> 상위 3개 문제 + 기존 대안. **설문 실측([`../04-survey.md`](../04-survey.md) §9)으로 우선순위 보정.**

### 상위 3대 문제 (자기보고 순위)

1. **손실 공포 / 손실 경험 (30.4%, 1위)** — "무서워서 못 한다 / 또 잃을까 봐". 신규 투자자 60%가 손실 경험([`../02-persona.md`](../02-persona.md) §2).
2. **어디서부터 시작·공부할지 모름 (25.0%) = 무엇을 믿을지 모름 (25.0%, 공동 2위)** — 미진입자 54%가 "방법 몰라서" 못 시작, 검색하면 광고·리딩방.
3. **(근본) 뉴스·공시를 봐도 독해 불가** — 직접 호소는 3.6%로 낮지만, 1·2위의 **뿌리**(리포트 46%·공시 21%만 읽음). → 표면 문제(손실·시작·신뢰)로 후킹하고 독해로 해결.

### 기존 대안 (Existing Alternatives)

| 대안 | 한계 |
|---|---|
| 유튜브/쇼츠·커뮤니티(54%·21%) | 검증 안 됨, 광고·리딩방 혼재 |
| 토스·증권사 앱(25%) | 거래 중심, 학습은 휘발성 |
| 어피티 뉴스레터(57%) | 일괄 발송, 주식 특화·개인화 약함 |
| 핀트(일임) | 대행이라 **학습이 안 남음** |
| 책·강의 | 지루해서 이탈 |
| 리딩방/유료추천 | 사기·고비용 |

> **[추론]** "대안이 많은데 왜 안 풀리나?" → 어느 것도 **개인화된 정확한 쉬움 + 학습 자산화**를 못 준다([`../01-benchmarking.md`](../01-benchmarking.md) §6). 이 공백이 곧 문제-솔루션 적합의 기회.

## 3. 솔루션 (Solution) — 린 특유 ★

> 각 문제에 대응하는 **최소 기능**만. (상세 기능·우선순위는 [`06-business-model-canvas.md`](06-business-model-canvas.md) §7, 향후 화면설계서)

| 문제 | 솔루션(MVP) | 근거 |
|---|---|---|
| ① 손실 공포 | **리스크·맥락 동반 해설** + 안전한 출발점(왕초보 트랙) | "정확한 쉬움"([`../01-benchmarking.md`](../01-benchmarking.md) §5) |
| ② 시작점 모름 | **내 종목 맞춤 아침 브리핑**(하루 5분 핵심) | 저니 P0([`../03-user-journey.md`](../03-user-journey.md) §7) |
| ③ 독해 불가 | **이슈 도슨트**: 뉴스·공시를 쉬운 말로 + **질문형 용어풀이** | 핵심 가치([`../02-persona.md`](../02-persona.md) §6) |
| 신뢰 | **출처·검증 시그널**(모든 해설에 출처) | 신뢰 게이트(저니 ①) |
| 리텐션 | **단계적 학습 커리큘럼 · 손실 복기** | 한회복 리텐션 |

> **[추론]** MVP는 "**내 종목 뉴스 1건을 출처와 함께 쉬운 말로 해설**"이라는 단일 가치부터 검증한다(저니 ③ 가치 체감 = 전환점).

## 4. 고유 가치 제안 (UVP) + 하이레벨 콘셉트

- **UVP:** **"시장 독해를 대신 해드립니다 — 내 종목 뉴스·공시를 출처 달린 쉬운 말로 풀어, 스스로 판단하게."**
- **하이레벨 콘셉트(High-Level Concept):** "**주식판 어피티 + 개인화**" / "**핀트의 반대 — 떠먹여주는 대신 읽는 법을 가르친다.**"
- **실측 근거:** F3에서 "독해 대행" **62.5%** 선호([`../04-survey.md`](../04-survey.md) §9). (상세 차별점 4가지는 [`06-business-model-canvas.md`](06-business-model-canvas.md) §2)

## 5. 고객군 (Customer Segments) + 얼리어답터

- **타깃:** 2030 주린이(경력 1년 안팎·소액·공시/리포트 못 읽음). 상세 6페르소나는 [`../02-persona.md`](../02-persona.md).
- **얼리어답터(Early Adopters) — 린 특유 ★:**
  - **김주린(28)** — 첫 손실 경험 + 학습 의지. 가장 전형적 핵심 타깃.
  - **한회복(29)** — 손실 후 "제대로 배우자". 문제 의식이 가장 강해 **초기 입소문·검증에 최적**.
- **안티 페르소나:** 전문 투자자·완전 위임 선호자·PB 고객(겨냥 안 함).

> **[추론]** 얼리어답터를 **한회복형(문제 절감이 절실한 손실 경험자)**으로 좁히면, "학습이 남는다"는 가치가 가장 잘 증명되고 초기 리텐션 신호를 빠르게 얻는다.

## 6. 채널 / 수익원 / 비용 구조 (BMC 동일 — 요약)

- **채널:** 숏폼/릴스(54%)·뉴스레터 제휴(57%)·앱 푸시·추천 루프·SEO → 상세 [`06-business-model-canvas.md`](06-business-model-canvas.md) §3.
- **수익원:** 무료 55.4% 전제 → 프리미엄 구독(월 4,600~6,900) + 광고/브랜디드 + B2B. SOM 3년차 약 7.7억([`05-market-size.md`](05-market-size.md) §4) → 상세 §5.
- **비용 구조:** LLM API(최대 동인)·데이터 API·인프라·인건비·마케팅·법무. 단위경제는 [`06-business-model-canvas.md`](06-business-model-canvas.md) §11 → 상세 §9.

## 7. 핵심 지표 (Key Metrics) — 린 특유 ★

> 해적 지표(AARRR)로 정리하고, 초기 단계의 **OMTM(One Metric That Matters)**을 지정한다.

| 단계 | 지표 | 메모 |
|---|---|---|
| 획득(Acquisition) | 가입 전환율 · CAC | 채널 효율(§6) |
| 활성화(Activation) | **첫 독해 "이해됐다" 도달률(Aha)** | 저니 ③ 가치 체감 |
| 유지(Retention) | **D7 / D30 리텐션 · 아침 브리핑 개봉률** | ← **초기 OMTM** |
| 매출(Revenue) | 무료→유료 전환율 · ARPU · LTV | 수익화 가설 |
| 추천(Referral) | 공유율 · 추천 가입 | 추천 루프(저니 ⑥) |
| 정렬(가치) | 해설 이해도 자가평가 · 생존율 | 거래유도 KPI 지양(로빈후드 반면교사) |

> **초기 OMTM = 리텐션(D7/D30).** "정말 이해된다"가 **습관**으로 이어지는지가 가장 강한 PMF 신호다. 매출·바이럴은 리텐션이 증명된 뒤 본다.

## 8. 불공정 우위 (Unfair Advantage) — 린 특유 ★

> "돈이나 시간으로 쉽게 살 수 없는" 진짜 해자만 적는다.

| 우위 | 모방 가능성 | 비고 |
|---|---|---|
| **1차정보 + 도메인 특화 해설 파이프라인 + 개인화 데이터 누적의 결합** | 낮음(시간 필요) | 쓸수록 강해지는 해자 |
| **학습 자산화** — 사용자에게 독해력이 남음 | 구조적 차별 | 핀트(대행)는 원리상 불가 |
| **"정확한 쉬움"**(리스크·맥락 동반) | 중 | 토스/로빈후드 단순화 부작용과 차별 |
| **타깃 신뢰 · 교육 포지션**(출처·비거래유도) | 낮음 | 거래수수료 모델과 이해 충돌 안 함 |

> ⚠️ **우위가 아닌 것:** LLM(Gemini)·벡터DB·인프라는 누구나 쓸 수 있어 **불공정 우위가 아니다**. 차별은 **데이터·파이프라인 품질과 신뢰**에서 나온다([`06-business-model-canvas.md`](06-business-model-canvas.md) §6).
