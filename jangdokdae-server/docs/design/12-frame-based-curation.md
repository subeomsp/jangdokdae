# 12. frame 기반 노출 큐레이션 설계 (무엇을·어떤 순서로 보여줄까)

> **작성자** Lee subeom · **작성일** 2026-06-19
>
> **범위** 생성된 콘텐츠(`issue_docent`)를 **전부 노출하지 않는다**. 분류(scope×frame)와 별개로, **frame 관점에서
> "무엇을·어떤 우선순위로 사용자에게 노출할지"**를 정하는 큐레이션 정책. 개인화(관심 종목·섹터)를 **전제**로 하되,
> ① 거시 이슈는 반드시 노출하고 ② frame엔 맞지만 가치 낮은 잡뉴스(예: 채용 발표)는 노출 단계에서 거른다.
>
> **핵심 결정**: 노출 결정 3계층(relevance 필터 → **frame 노출가치 게이트** → 개인화 슬롯) · 잡뉴스는 *생성 후*
> `display_worthy=false`로 비파괴 제거 · 거시(시장 전체)는 'C 슬롯'으로 노출 보장 · 슬롯·거리 모델은 [08 §4](./08-pain-to-content-pipeline.md) 계승.

## 1. 목적과 위치

분류 파이프라인([10](./10-content-pipeline-implementation.md))은 클러스터마다 `issue_docent`(콘텐츠)를 만든다. 하지만
**만든 것을 다 보여주면 레거시 미디어와 다를 게 없다.** 주린이에게는 "오늘 이 3~5개만 읽으면 된다"가 약속이다([08 §4](./08-pain-to-content-pipeline.md)).

이 문서는 **노출 결정**을 다룬다 — *분류가 무엇인가*(scope×frame, 10번)가 아니라 *그중 무엇을 띄울까*. 두 축이 있다:
- **scope/거리(개인화)** — "이게 나와 얼마나 가까운가". [08 §4](./08-pain-to-content-pipeline.md)의 A/B/C/D 슬롯이 담당.
- **frame(노출가치)** — "이 *읽는 유형*이 띄울 만한가, 어떤 우선순위인가". 08은 이를 *2차 기준*으로 짧게만 언급 → **본 문서가 상세화**한다.

전제: **개인화는 필수다.** 우리 서비스는 가입 시 **관심 종목·관심 섹터**를 받는다(온보딩). 따라서 큐레이션은
"개인화된 후보 위에 frame 우선순위를 얹는" 구조다. 단, 개인화만으로는 *내 종목과 무관한 거시 이슈*가 탈락하므로,
거시는 frame 기준으로 **별도 보장**한다(§3·§5).

## 2. 노출 결정 3계층

노출은 단일 판단이 아니라 **3계층 필터·정렬**이다. 위에서 아래로 좁혀진다.

```
[전체 수집 뉴스]
  │  ① relevance 필터 (분류 단계, 이미 구현 → eval/04)
  │     비투자성(홍보·사회공헌·ESG·마케팅·교육·부고/인사) = is_investment_relevant=false
  │     → 콘텐츠 생성 자체를 skip (issue_docent 안 만듦)
  ▼
[issue_docent 생성됨 = 투자 관련 뉴스]
  │  ② frame 노출가치 게이트 (생성 후, 본 문서 §3·§4)
  │     - frame엔 맞지만 가치 낮은 건(채용·루틴 PR성 PLAN·약한 PRICE) → display_worthy=false (숨김)
  │     - frame별 노출 우선순위 부여(§3)
  ▼
[노출 후보 = 띄울 만한 콘텐츠]
  │  ③ 개인화 슬롯 배치 (08 §4)
  │     관심 종목/섹터와의 거리(A/B) + 거시 보장(C) + 학습(D) → 하루 3~5개
  ▼
[사용자에게 뜨는 3~5개 + "왜 골랐나" 한 줄]
```

- **①과 ②의 차이**가 핵심이다. ①은 "**투자 뉴스가 아예 아님**"(보여줄 이유 없음), ②는 "**투자 뉴스는 맞지만 굳이 볼 필요 없음**"
  (예: "○○증권, 신입 공채 100명" — PLAN이긴 하나 투자 판단엔 무의미). ①은 분류 전에 끊어 비용을 아끼고, ②는 생성 후 노출만 끈다.
- ②를 *생성 후*에 두는 이유: 분류(PLAN 등)는 그대로 보존(가역·재평가 가능)하고, 노출만 비파괴로 제어하기 위함.

## 3. frame 노출가치 등급

같은 후보가 슬롯·상한(3~5개)을 두고 경합하면 **frame이 우선순위를 가른다**([08 §4](./08-pain-to-content-pipeline.md) "거리 다음은 frame" 계승·확장).

| frame | 노출가치 | 근거 |
|-------|---------|------|
| `INCIDENT` (악재) | **가산(↑)** | 손실 공포(P2)가 가장 큰 페인 — 내 종목·내 업종 악재는 동점이면 먼저 올린다 |
| `EARNINGS`·`PLAN`·`POLICY`·`TREND`·`OPINION` | 중립 | 중요도 점수(volume/velocity 등)로 정렬 |
| `PRICE` (주가만 움직임) | **최하(↓)** | 뚜렷한 원인 없는 가격 변동 → 원인 있는 frame과 경합하면 밀린다('따라 사기' 유발을 선택 단계에서도 차단) |

**거시 보장(개인화와 독립)**: scope=`시장 전체`(금리·환율·세제·지수·지정학, 주로 `POLICY`/거시 `TREND`)는 어느 사용자의 보유와도
무관하지만 "모두의 돈 얘기"라 반드시 1건은 노출한다 → §5의 **C 슬롯**으로 항상 1자리 확보. 개인화가 강해도 거시가 탈락하지 않게 하는 안전장치.

> 노출가치 등급은 frame "종류"의 기본 가중치다. 개별 건의 *품질* 컷오프는 §4(게이트)가 따로 본다.

## 4. frame 내 노출가치 게이트 (잡뉴스 제거)

relevance 필터(①)를 통과해 콘텐츠가 생겼더라도, **frame엔 맞지만 띄울 가치가 낮은** 건이 있다. 이를 `display_worthy=false`로 숨긴다.

| 패턴 | frame | 왜 가치 낮나 |
|------|-------|-------------|
| 채용·공채 발표 | PLAN | 미래 약속이긴 하나 매출·주주가치 경로가 약함 — 투자 판단에 무의미 |
| 루틴 PR성 계획 | PLAN | MOU·업무협약·소액 투자 등 "했다"는 사실뿐, 돈 되는 경로 불명 |
| 원인·움직임 약한 가격 | PRICE | 소폭 등락·단순 시황 나열 — 읽어도 행동 근거 없음 |
| 알맹이 없는 단순 동정 | 다수 | 수상·행사·의례적 코멘트 등 |

- **relevance 필터(①)와의 차이**: ①의 비투자성(부고·사회공헌)은 *투자와 무관*이라 분류 전 끊는다. ②의 잡뉴스는 *투자 관련이긴 하나 가치 낮음* —
  분류는 남기고(PLAN 등) 노출만 끈다. "채용"이 대표 예: 회사 활동이라 PLAN으로 분류되지만 주린이가 굳이 볼 필요는 없다.
- **판정 기준(개념)**: "이 건이 독자의 *투자 판단·학습*에 보탬이 되는가?" 아니오면 `display_worthy=false` + 사유 기록.
  보수적으로(애매하면 노출) 두되, 명백한 잡뉴스만 끈다. 판정 주체는 §7(생성 시 LLM 신호 또는 규칙)에서 다룬다.

## 5. 개인화 전제 + 슬롯 합성

개인화(관심 종목·섹터, 가입 시 수집)를 전제로 [08 §4](./08-pain-to-content-pipeline.md) 슬롯을 채운다. **노출 후보 = ① 통과 ∧ ② `display_worthy=true`.**

| 슬롯 | 거리 | 개수 | frame 큐레이션이 하는 일 |
|------|------|------|------------------------|
| A. 내 주식에 생긴 일 | 관심 **종목** 직접 | 0~2 | 후보를 frame 노출가치+중요도로 정렬(INCIDENT 우선, PRICE 후순위) |
| B. 내 주식에 번질 일 | 관심 **섹터** 동일 | 0~1 | 같은 frame 게이트·정렬 적용 |
| C. 모두의 돈 얘기 | 거시(시장 전체) | 1 | **거시 보장** — 개인화와 무관히 오늘 최대 거시 이슈 1건(§3) |
| D. 읽으면 느는 일 | 학습 | 1 | 사용자가 **아직 안 본 frame**을 골라 7개 읽는 법을 고루 학습 |

- **거리(슬롯)가 1차, frame이 2차**다(08 계승). 거리가 "왜 나한테 떴나"의 신뢰를 주고, frame이 같은 슬롯 내 우선순위·다양성을 정한다.
- **동일 frame ≤ 2/일** — 하루치에 같은 읽는 유형이 몰리지 않게(08 §4). 자리가 남으면 안 나온 frame으로 채워 D슬롯 학습 효과를 넓힌다.
- 빈 슬롯 처리(08 §4.3 계승): 관심 종목 뉴스가 없으면 A를 C·D로 채우고 "내 종목은 오늘 다룬 이슈 중엔 없었어요"라고 정직하게 알린다.

## 6. 일일 노출 셋 구성 (개념 알고리즘)

```
후보 = issue_docent 중 (is_investment_relevant=true) ∧ (display_worthy=true)         # ①②
A ← 관심종목 매칭 후보 → (INCIDENT 우선, PRICE 후순위, 중요도) 정렬 → 상위 0~2
B ← 관심섹터 매칭(아직 안 뽑힘) → 같은 정렬 → 0~1
C ← scope=시장전체 후보 중 중요도 1위 → 1   (거시 보장)
D ← 사용자가 아직 안 본 frame 중 1건(학습)  → 1
컷: 총 3~5개, 동일 frame ≤ 2, PRICE는 원인 있는 후보에 밀림
각 항목에 "왜 골랐나" 한 줄 부착(08)                                              # 신뢰
```

- 미진입자(관심 종목 없음): A·B가 비어 C 1~2 + D 2~3로 5개 — D가 입문 코스(08 §4.3).
- 정렬 신호 = `news_cluster.importance`(volume/velocity, MVP) + frame 노출가치 가중. importance 고도화는 05 §11/후속.

## 7. 구체 사례 (실데이터 worked example)

2026-06-19 라이브 적재분(노출 후보 78 + relevance 제외 8)에 **제안 정책을 적용하면** 무엇이 빠지고 무엇이 우선되는지 보인다.
①relevance 필터는 이미 구현·반영([eval/04](../evaluation/04-classification-improvement-history.md))됐고, ②노출가치 게이트·개인화 슬롯은 **미구현**이라 아래 §7.2~§7.4는 *정책 시뮬레이션*이다.

| 5 | PLAN | 예탁원, 증권박물관 주말 교육프로그램 운영 | 교육/행사 |

### 7.2 제외 — ② frame 노출가치 게이트 (투자엔 맞지만 가치 낮음)

**대표 사례 [19] "[크립토Now] 코인원, 서비스 경쟁력 강화 나선다…프로덕트 직군 채용"**
- 현재 상태: `is_investment_relevant=true` → PLAN으로 분류·콘텐츠까지 **생성돼 노출 후보 78건에 포함**돼 있다(①을 통과).
- 문제: '채용'은 미래 약속(PLAN)이긴 하나 매출·주주가치로 가는 경로가 약해 **주린이 투자 판단엔 무의미** → §4 기준 `display_worthy=false` 대상.
- **①과 ②의 차이가 이 한 건에 드러난다**: ①(위 8건)은 *투자 무관*이라 콘텐츠조차 안 만든다. ②([19])는 *투자 관련(PLAN)*이라
  콘텐츠는 있으나 **노출만 끈다**. 분류(PLAN)·콘텐츠는 보존하고 `display_worthy`만 false.
- 보조 후보(루틴 PR성 — 신규성·돈 되는 경로가 약하면 게이트 검토): [287] "신한 슈퍼SOL 공개", [288] "한국성장금융 출범 10주년".

### 7.3 우선순위 — 미진입자(관심 종목 없음) 오늘의 5개

**핵심 관찰: importance가 대부분 동률(0.057 등)이라, 실제 순위는 frame 노출가치 가중이 가른다.**

| 슬롯 | 뽑힌 이슈 | frame | imp | 왜 |
|---|---|---|---|---|
| **C 거시 보장** | [29] 증시 승부처는 '금리' | POLICY/시장전체 | 0.400 | 오늘 최대 거시 — 개인화 무관 1자리 확보 |
| 본문 | [320] 금감원, 미래에셋 검사 연장 | INCIDENT/회사 | 0.076 | **INCIDENT 가산**(손실 공포) — 동점이면 우선 |
| 본문 | [332] "소외된 지금이 매수 기회"…에코프로 | OPINION/회사 | 0.171 | 전문가 평가 — 중요도 상위 |
| 본문 | [327] 평화협정에도 방산주↑…방공망 수출 | TREND/회사 | 0.057 | 업황 변화 |
| **D 학습** | [303] 젠큐릭스, 293억 국가 과제 참여 | PLAN/회사 | 0.076 | 아직 안 나온 frame(PLAN) — 읽는 법 학습 |

- **PRICE 강등 실증**: [334] "외국인 매수…코스피 신고가"는 **importance 0.400(공동 최상)**이지만 *원인 없는 PRICE*라 **5개에서 밀린다**.
  원인 있는 frame(POLICY·INCIDENT·OPINION·TREND·PLAN)이 먼저 차고, '따라 사기' 유발 노이즈는 선택 단계에서 빠진다(§3).
- **frame 다양성**: 위 5개는 POLICY·INCIDENT·OPINION·TREND·PLAN로 전부 다른 읽는 법 → 동일 frame ≤2 충족, D슬롯이 학습 효과를 넓힌다.

### 7.4 개인화 사용자 변형 (관심: 삼성전자·SK하이닉스 / IT 섹터)

거리(개인화)가 1차 기준이라, **내 종목이면 frame이 낮아도(PRICE) A슬롯으로 노출**된다(08 §4).
- **A 내 종목**: [334] 삼성전자·SK하이닉스 매수(PRICE) — §7.3에선 강등됐지만, *내 종목*이라 "내가 가진 ○○ 소식"으로 A에 올라온다.
  [331] SK그룹·하이닉스 합산시총.
- **B 내 섹터(IT)**: IT 섹터로 분류된 이슈 1건.
- **C 거시**: [29] 금리(동일 보장). **D 학습**: 아직 안 본 frame.

→ 같은 [334](PRICE)라도 *미진입자에겐 강등·제외*, *보유자에겐 A슬롯 노출*. **거리(개인화) → frame(노출가치)** 2단 정렬이 사용자마다 다른 결과를 만든다.

## 8. 데이터·구현 메모 (후속)

본 문서는 **설계**다. 구현(별도 PR)에서 정할 것:
- **`display_worthy` 신호를 어디 둘까**: (a) 생성 시 LLM이 함께 판정해 `issue_docent`에 컬럼으로 적재(분류처럼), 또는
  (b) 노출(서빙) 시점에 규칙/별도 판정으로 계산. relevance 필터(`is_investment_relevant`)와 대칭으로 (a)가 일관적 —
  생성 단계 또는 분류 결과(frame+근거)로부터 1차 규칙 + 필요 시 LLM 보조.
- **`is_published`와의 관계**: 현재 `issue_docent.is_published`(기본 false)는 발행 게이트. `display_worthy`(노출가치)와
  분리하거나, 발행 기준에 흡수할지 결정(노출가치 false면 미발행 등). [06 §21](./06-news-analysis-design.md) "발행 기준" 미결 항목과 연계.
- **개인화 데이터 모델 미구현**: 관심 종목·섹터(온보딩)·사용자 보유 ORM·서빙 API([users/interests 라우터](../../app/api/routers/) 빈 상태)는
  아직 없다 → 슬롯 A/B 실서빙은 이 모델 구현이 선행돼야 한다. C(거시)·D(학습)는 개인화 없이도 동작 가능.

## 9. 한계·후속

- **frame 노출가치·잡뉴스 기준은 휴리스틱** — 실데이터·사용자 반응(읽음률·이탈)으로 교정 필요. §4 게이트의 false 사유를 로깅해 보정.
- **importance 신호 MVP** — volume/velocity만 반영(sentiment·entity는 0, 05 §11). 정렬 품질은 NER·감성 연결 후 개선.
- **개인화 미구현** — 슬롯 A/B는 관심 종목·섹터 데이터 모델·서빙이 선행돼야 실동작.
- **거시 보장 vs 다양성 충돌** — C(거시) 강제 1자리와 동일 frame ≤2가 작은 노출 수(3개)에서 경합할 수 있어, 우선순위 규칙을 구현 시 명문화.
