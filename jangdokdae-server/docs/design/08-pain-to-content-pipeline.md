> **이 문서가 답하는 것**

> 3. 콘텐츠는 **어떻게 구성**되고, 각 콘텐츠는 **어떤 고민을 어떻게** 풀까
> 4. 만드는 사람 입장에서, 분류별로 뉴스 말고 **어떤 자료를 왜 더 끌어와** 콘텐츠를 채울까

## 0. 한눈에 보기

장독대는 주린이가 "오늘은 이것만 읽으면 된다"를 받도록, 들어온 뉴스를 분류하고 → 보여줄 것을 고르고 → 자료를 붙여 → 콘텐츠로 만든다.

```
고민(P1~P6)
   │
   │   ① 뉴스를 나눈다       누구 얘기(scope) × 어떻게 읽나(frame)        → §2, §3
   │
   │   ② 오늘 보여줄 걸 고른다  하루 3~5개, 나와 가까운 순서 + 왜 골랐는지   → §4
   │
   │   ③ 자료를 더 붙인다      frame별로 공시·재무·거시·주가·보고서        → §5
   │
   │   ④ 콘텐츠로 만든다       해설 4단 + 콘텐츠 10종                     → §6
   │
   └── ⑤ 고민을 푼다          어떤 콘텐츠가 어떤 페인을 어떻게            → §7
```

## 1. 우리가 푸는 고민 (페인포인트)

타깃은 **김주린**(28세, 투자 8개월, 뉴스 헤드라인은 읽지만 공시·리포트는 못 읽는 직장인) 같은 주린이다. 이들의 고민은 6개다.

| ID | 고민 | 종류 | 사용자가 입으로 말하나? |
| --- | --- | --- | --- |
| **P1** | 뉴스·공시·리포트를 못 읽음 (독해) | 기능 | ✕ 자기도 모름 (설문 2.6%, 최하위) — **진짜 원인** |
| **P2** | 또 잃을까 무섭다 / 이미 잃었다 | 감정 | ○ 1위 (35.9%) — **겉으로 호소** |
| **P3** | 어디서부터 시작할지 모름 | 기능 | ○ 2위 (28.2%) — **겉으로 호소** |
| **P4** | 검색하면 광고·리딩방뿐, 못 믿음 | 기능·감정 | △ |
| **P5** | "그것도 몰라?" 시선이 무섭다 (위축) | 감정 | △ 잠재 |
| **P6** | 분석할 시간이 없다 | 시간 | ○ 3위 (17.9%) |

**핵심.** 사용자는 손실(P2)·시작점(P3) 언어로 말을 건다. 정작 진짜 원인인 독해(P1)는 자기도 모른다. 그래서 우리는 **손실·시작점 언어로 말 걸고, 그 안에서 독해를 떠먹인다.**

## 2. 뉴스를 나누는 두 가지 질문

뉴스 한 건이 들어오면 두 가지를 정한다.

### 2-1. 누구 얘기인가 — scope (3가지)

뉴스에 고정된 성질이다. 누가 봐도 같다.

| scope | 뜻 | 예시 |
| --- | --- | --- |
| **회사** | 특정 기업 한 곳 얘기 | "삼성전자, 3분기 영업이익 9.2조" |
| **업종·테마** | 산업·테마 전체 얘기 | "D램 값 반등, 메모리 업황 회복" |
| **시장 전체** | 모두의 돈에 닿는 얘기 (금리·환율·세금·지수) | "한국은행 기준금리 0.25%p 인하" |

### 2-2. 어떻게 읽어야 하나 — frame (7가지)

뉴스를 어떤 눈으로 읽어야 하는지다. 이게 해설 구조와 푸는 고민을 정한다. **상세는 §3.**

> **원인 우선 규칙.** 주가 움직임은 늘 결과다. 원인(실적·사건·정책 등)을 먼저 찾아 frame을 정하고, 뚜렷한 원인이 없을 때만 "주가" frame이다. 예: "실적 발표에 급등"은 주가 frame이 아니라 실적 frame이다.

### 국내/해외는 분류 축이 아니다

국내든 해외든 **읽는 법(frame)도 읽을 개수도 똑같다.** 실적은 실적이고 악재는 악재다. 해외 뉴스라고 더 보여주거나 덜 보여줄 이유가 없다. 바뀌는 건 **해설 제목 문구 몇 글자뿐**이다 ("얼마나 영향?" → "우리 시장엔 어떻게 올까?"). 그래서 국내/해외는 **분류 축이 아니라 문구만 바꾸는 표시(flag)**로 둔다.

---

## 3. 읽는 유형 7가지 (frame) 상세

frame 이름은 **내부에서 쓰는 코드**(안 바뀌는 영어)와 **사용자에게 보이는 이름**(직관적인 우리말)을 따로 둔다.

각 유형마다 **해설 4단**이 고정돼 있다. 3단은 항상 **★오해 방지**다 — "좋은 숫자 = 무조건 좋음" 같은 순진한 해석을 막고, 답을 단정하지 않고 사용자가 판단하게 한다.

### ① EARNINGS — "실적이 나왔어요"

**무슨 뉴스**: 매출·영업이익 등 회사 성적표 발표

| 단 | 질문 |
| --- | --- |
| 1단 | 무슨 일이에요 (가장 크게 좋아지거나 나빠진 숫자는?) |
| 2단 | 시장 예상과 비교하면 (잘한 건가, 못한 건가?) |
| 3단 | ★이게 계속될까요 (한 번 반짝인가, 흐름인가?) |
| 4단 | 앞으로 뭘 보면 될까요 |

**푸는 고민: P1(독해).** "9.2조"가 큰지 작은지 모르는 사람에게 "예상과 비교하면"으로 판단 기준을 떠먹인다.

### ② INCIDENT — "악재가 생겼어요"

**무슨 뉴스**: 사고·소송·리콜·화재·제재 등 나쁜 일

| 단 | 질문 |
| --- | --- |
| 1단 | 무슨 일이에요 |
| 2단 | 얼마나 영향을 줄까요 |
| 3단 | ★일시적인가요, 계속될 문제인가요 |
| 4단 | 앞으로 뭘 보면 될까요 |

**푸는 고민: P2(손실 공포).** 막연한 공포 대신 "얼마나/얼마 동안" 걱정할 일인지 크기를 알려준다.

### ③ PLAN — "새 계획을 발표했어요"

**무슨 뉴스**: 신제품·대규모 투자·수주·인수·주주환원 등 회사의 약속·계획

| 단 | 질문 |
| --- | --- |
| 1단 | 무슨 일이에요 |
| 2단 | 기존과 뭐가 다른가요 (약속이 매출로 가는 길은?) |
| 3단 | ★진짜 돈이 되려면 (남은 관문은?) |
| 4단 | 앞으로 뭘 보면 될까요 |

**푸는 고민: P4(신뢰).** 보도자료를 그대로 믿지 않게, "진짜 돈이 되려면 뭐가 더 필요한지"를 짚는다.

### ④ POLICY — "제도가 바뀌어요"

**무슨 뉴스**: 정부 정책·규제·금리·세금 등 규칙 변화

| 단 | 질문 |
| --- | --- |
| 1단 | 무슨 규칙이에요 |
| 2단 | 누구한테 이득·부담일까요 |
| 3단 | ★확정된 건가요, 검토 중인가요 |
| 4단 | 앞으로 뭘 보면 될까요 |

**푸는 고민: P3(시작점)·P6(시간).** 남 얘기처럼 느껴지는 큰 뉴스를 "그래서 내 돈엔 어떻게 오나"로 연결한다.

### ⑤ TREND — "업황이 달라져요"

**무슨 뉴스**: 산업 전체의 흐름·트렌드·수급 변화

| 단 | 질문 |
| --- | --- |
| 1단 | 무슨 변화예요 |
| 2단 | 왜 중요한가요 |
| 3단 | ★잠깐인가요, 계속될 변화인가요 |
| 4단 | 앞으로 뭘 보면 될까요 |

**푸는 고민: P1(독해)·P2(손실).** 추상적인 "업황 회복"을 내 종목까지 닿는 그림으로 풀어준다.

### ⑥ OPINION — "전문가가 평가했어요"

**무슨 뉴스**: 증권사 리포트·목표주가·투자의견 등 전문가의 판단
**예시 기사**

- "키움증권, 에코프로 목표가 60만원으로 하향…투자의견 '중립'"
- "KB증권, 현대모비스 목표가 75만→120만원 상향"

| 단 | 질문 |
| --- | --- |
| 1단 | 누가, 뭐라고 했어요 (목표가·투자의견·직전 대비) |
| 2단 | 지금 주가에서 얼마나 더 본 거예요 (현재가 대비 괴리율) |
| 3단 | ★이 숫자, 믿어도 될까요 (목표가를 떠받치는 가정이 타당한가) |
| 4단 | 앞으로 뭘 보면 될까요 |

**푸는 고민: P4(신뢰)·P1(독해).** "목표가 60만원"이라는 숫자를 그대로 믿지 않게, 현재가 대비 거리(괴리율)와 그 목표가를 떠받치는 가정을 같이 보여준다.

### ⑦ PRICE — "주가만 움직였어요"

**무슨 뉴스**: 뚜렷한 원인 없이 가격이 출렁인 경우 (원인이 있으면 위 ①~⑥으로 간다)

| 단 | 질문 |
| --- | --- |
| 1단 | 무슨 일이에요 (얼마나 움직였나?) |
| 2단 | 왜 움직였대요 |
| 3단 | ★진짜 이유일까요, 끼워 맞춘 걸까요 |
| 4단 | 앞으로 뭘 보면 될까요 |

**푸는 고민: P2(따라 사기).** "이유 없이 급등"에 휩쓸려 추격매수하지 않게, 이유가 진짜인지 의심하게 한다.

### 정리표

| 내부 코드 | 사용자에게 보이는 이름 | 무슨 뉴스 | 푸는 고민 |
| --- | --- | --- | --- |
| EARNINGS | 실적이 나왔어요 | 매출·이익 성적표 | P1 |
| INCIDENT | 악재가 생겼어요 | 사고·소송·리콜·제재 | P2 |
| PLAN | 새 계획을 발표했어요 | 신제품·투자·수주·인수·주주환원 | P4 |
| POLICY | 제도가 바뀌어요 | 정책·규제·금리·세금 | P3·P6 |
| TREND | 업황이 달라져요 | 산업 흐름·수급 변화 | P1·P2 |
| OPINION | 전문가가 평가했어요 | 증권사 리포트·목표주가·의견 | P4·P1 |
| PRICE | 주가만 움직였어요 | 이유 불명 급등락 | P2 |

### 적용 범위 점검 — 들어오는 뉴스를 다 담나

수집 피드는 한국 주요 경제·증권 매체의 종합 증권 뉴스다. 종목·시황·정책·리포트가 섞여 들어온다. 위 7개 유형은 그중 **개별 종목·업종·정책·전문가 의견 기사 대부분을 담는다.** 다만 자주 들어오지만 7개로 바로 안 맞는 유형이 있어, 처리 규칙을 둔다.

| 자주 들어오는데 안 맞는 유형 | 예시 | 처리 |
| --- | --- | --- |
| **시황·마감 종합** (지수·수급 종합) | "코스피 2,600 마감…외국인 5천억 순매수" | 개별 해설 대상 아님. **시장 scope로 묶어 하루 1건 요약**(§4 C슬롯)으로만 |
| **자본 변동** (유상증자·전환사채) | "○○ 1,500억 유상증자 결정" | 희석=악재 성격 → **INCIDENT**로 배정 (배당·자사주 같은 주주환원은 PLAN) |
| **안내·정보성** (거래정지·배당락·IPO 일정) | "○○ 내일 배당락" | 해설할 게 없음 → **필터(노출 안 함)** |
| **기획·인터뷰·칼럼** | "○○ 대표 인터뷰" | 사실 보도 아님 → 대부분 필터, 업황 통찰이면 TREND |

즉 **7개 frame + 4개 처리 규칙**으로 들어오는 뉴스의 큰 비중을 흡수한다. 어디에도 안 맞고 가치도 낮은 기사는 억지로 분류하지 않고 필터한다.

## 4. 하루에 몇 개를, 왜 보여줄까

### 왜 하루 3~5개인가

레거시 미디어는 하루 수백 건을 쏟아낸다. 주린이는 **그중 뭘, 얼마나 읽어야 할지 모른다.** 다 읽으면 기존 미디어와 다를 게 없고, 안 읽으면 불안하다. 우리의 약속은 단순하다.

> **"오늘은 이 3~5개만 읽으면 돼요. 각각 왜 골랐는지도 알려드릴게요."**

개수를 3~5개로 묶은 이유:

- **너무 적으면(1~2개)** 내 종목·시장·공부를 한 번씩 담지 못한다.
- **너무 많으면(6개 이상)** 레거시 미디어와 차이가 없고, 부담돼서 안 읽는다.
- **3~5개**면 출퇴근길 5분 안에 소화되면서, 내 종목 → 시장 → 공부를 한 번씩 커버한다.

### 어떻게 3~5개를 채우나 — 나와 가까운 순서(거리)

같은 뉴스라도 사람마다 거리가 다르다. 위에서부터 검사해 **처음 걸리는 칸 하나**를 준다. 그리고 가까운 칸부터 슬롯을 채운다.

| 슬롯 | 거리 | 개수 | 고르는 기준 | **사용자에게 뜨는 "왜 골랐나" 한 줄** |
| --- | --- | --- | --- | --- |
| 1~2 | **A. 내 주식에 생긴 일** | 0~2 | 보유 종목 뉴스 중 중요한 순 | "내가 가진 ○○ 소식이에요" |
| 3 | **B. 내 주식에 번질 일** | 0~1 | 보유·관심과 같은 업종 | "내 ○○와 같은 업종이라 번질 수 있어요" |
| 4 | **C. 모두의 돈 얘기** | 1 | 오늘 시장 최대 이슈 | "오늘 시장 전체가 움직인 일이에요" |
| 5 | **D. 읽으면 느는 일** | 1 | 아직 안 배운 읽는 유형 | "오늘 읽기 연습으로 좋은 뉴스예요" |

**"왜 보여주나"를 항상 한 줄로 붙이는 게 핵심이다.** 사용자가 "이게 왜 나한테 떴지?"를 묻지 않게 하고, 안 읽어도 될 뉴스를 솎아냈다는 신뢰를 준다.

### 칸이 빌 때 / 사람이 다를 때

- **내 종목 뉴스가 없는 날**: A 슬롯을 C·D로 채우되, **"내 종목은 오늘 다룬 이슈 중엔 없었어요"**라고 정직하게 알린다. "뉴스가 없다"가 아니라 "우리가 다룬 것 중엔 없었다"고 말한다 (추측·억지 생성 금지).
- **아직 종목이 없는 사람(미진입자)**: A·B가 늘 비므로 **C 1~2개 + D 2~3개**로 5개를 채운다. 이때 D칸(읽으면 느는 일)이 곧 **입문 코스**가 된다.
- **시간이 없는 사람**: 같은 3~5개를 **한 줄씩 아침 브리핑** 모양으로 압축해 보여준다. 분류는 그대로, 모양만 바꾼다.

### 무엇을 먼저·무엇을 자르나 — 거리 다음은 frame

거리(슬롯)는 **무엇을·왜 보여줄지와 개수 상한**을 정하는 1차 기준이다("왜 나한테 떴나"의 신뢰가 거기서 나온다). 같은 슬롯에 후보가 몰리거나 그날 이슈가 상한(3~5개)을 넘으면, **frame이 2차 기준**으로 우선순위와 다양성을 정한다.

- **중요도 점수 (1차 정렬)** — 같은 이슈를 다룬 기사 수(얼마나 많이 다뤄졌나) + 관련 공시·재무가 붙는지. 많이 다뤄지고 공식 자료까지 있는 이슈가 위로 온다.
- **frame 가중치 (점수에 합산)**
  - **주가(PRICE)는 최하** — §2-2 "원인 우선 규칙"의 연장이다. 뚜렷한 원인 없는 가격 변동이라, 원인 있는 다른 frame과 경합하면 밀린다(따라 사기 유발을 선택 단계에서도 차단).
  - **악재(INCIDENT)는 가산** — 손실 공포(P2)가 가장 큰 페인이라, 내 종목·내 업종 악재는 동점이면 먼저 올린다.
  - 나머지(실적·계획·제도·업황·전문가)는 중립 — 위 중요도 신호로 정렬한다.
- **frame 다양성 (중복 제한)** — 하루 3~5개에 같은 frame이 몰리지 않게 **동일 frame은 최대 2개**까지. 자리가 남으면 아직 안 나온 frame으로 채워 D슬롯(읽으면 느는 일)의 학습 효과를 전체로 넓힌다.

> 예: 보유 종목이 없는 사용자는 A·B가 비어 그날 시장 이슈가 C·D로 몰린다. 7개 frame이 한 건씩 경합해도, **거리로 상한(≤5)을 자르고 → 중요도+frame 가중치로 'C: 오늘 최대 이슈'를 고르고 → 'D: 못 본 frame'을 학습용으로 돌린다.** 주가(PRICE) 노이즈는 원인 있는 이슈에 밀려 자연히 빠진다.

> **(2026-06-19, →[12](./12-frame-based-curation.md))** 여기 'frame 2차 기준'(노출가치·다양성·거시 보장·잡뉴스 제거)은
> [12번 frame 기반 노출 큐레이션](./12-frame-based-curation.md)에서 상세화했다(노출 결정 3계층·`display_worthy` 게이트 포함).

## 5. ★ frame별 추가자료 — 무엇을, 왜 붙이나 (만드는 사람 쪽)

해설 4단을 쓰다 보면 **뉴스 본문만으로는 답이 안 나오는 칸**이 생긴다. 그 빈칸을 우리가 이미 모으는 자료로 채운다. 핵심은 **"왜 이 자료가 없으면 안 되는가"**다.

### 5-1. 우리가 쓸 수 있는 자료 (이미 수집 중)

| 자료 | 가져오는 방식 | 들어있는 것 |
| --- | --- | --- |
| **공시 (DART)** | **RAG(벡터 검색)** + 접수번호 조회 | 회사 공식 발표 원문·종목코드 |
| **사업보고서** | **RAG(벡터 검색)** | 사업 내용·경영진단·감사의견 |
| **재무제표** | 키 조회 (회사·분기) | 매출·영업이익·순이익·자산 |
| **거시지표** | 키 조회 (종류·날짜) | 환율·금리·물가·통화량 |
| **주가** | 키 조회 (종목·날짜) | 시·고·저·종가·거래량·시총 |

> **방식 두 가지를 구분한다.**
> - **RAG(벡터 검색)**: "관련된 걸 찾아오는" 방식. 공시·사업보고서처럼 글 덩어리라, 뉴스와 의미가 비슷한 부분을 검색해 가져온다.
> - **키 조회**: "정해진 값을 정확히 꺼내오는" 방식. 재무·거시·주가처럼 숫자라, 회사·날짜만 알면 바로 꺼낸다. 글은 RAG로 찾고, 숫자는 키로 꺼낸다. 섞지 않는다.

### 5-2. frame별 매핑 + 왜 필요한가

| frame | 비는 해설 단 | 붙일 자료 (방식) | **왜 필요한가** | 푸는 고민 |
| --- | --- | --- | --- | --- |
| **실적** | 2단 "예상과 비교" | 재무제표(조회) + 사업보고서(RAG) | 뉴스엔 "9.2조"만 있다. 과거 실적과 비교해야 잘한 건지 알고, 보고서로 어느 사업부 숫자인지 맥락이 잡힌다. **없으면 숫자가 큰지 작은지 판단 불가.** | P1·P4 |
| **악재** | 2단 "얼마나 영향" | 공시 원문(RAG) + 주가(조회) | 뉴스는 "사고 발생"만 전한다. 공시로 회사 공식 입장·규모를, 주가 낙폭으로 시장이 얼마나 심각하게 보는지 가늠한다. **없으면 막연한 공포만 커진다.** | P2·P4 |
| **새 계획** | 2·3단 "진짜 돈 되나" | 공시 원문(RAG) + 재무제표(조회) | "1조 투자 발표"가 진짜일지는, 회사 체력(재무)으로 감당되는지·공시에 구체 일정이 있는지 봐야 안다. **없으면 보도자료를 그대로 믿게 된다.** | P4 |
| **제도** | 2단 "누가 이득" | 거시지표(조회) + 업종 매핑 | "금리 인하"가 내게 이득인지 부담인지는, 거시 흐름과 내 업종을 엮어야 나온다. **없으면 남 얘기로 흘려보낸다.** | P3·P6 |
| **업황** | 1·2단 "무슨 변화·왜 중요" | 동종 기업 보고서(RAG) + 거시(조회) | "메모리 업황 회복"은 그 업종 기업들·거시로 살을 붙여야 그림이 된다. **없으면 추상적 흐름으로만 남는다.** | P1·P2 |
| **전문가 의견** | 2단 "현재가 대비 괴리" + 3단 "믿어도 될까" | **주가(조회, 현재가→목표가 괴리율)** + 재무제표(조회) | "목표가 60만원"은 지금 주가와 얼마나 차이 나는지(괴리율), 근거가 재무로 말이 되는지 봐야 가늠한다. **현재가가 없으면 목표가가 공중에 뜬다.** | P4·P1 |
| **주가** | 2단 "진짜 이유?" | 공시·거시 교차 확인(RAG+조회) + 주가 | "이유 없이 급등"이 정말 이유 없는지 공시·거시를 뒤져 확인한다. 없으면 **"끼워 맞춘 이유"**라고 정직하게 쓴다. **안 뒤지면 추격매수를 부추긴다.** | P2 |

### 5-3. scope에 따른 보강

| scope | 우선 붙이는 자료 | 왜 |
| --- | --- | --- |
| 회사 | **사업보고서(RAG)** | 이 회사가 뭐 하는 곳인지, 해당 사업부 맥락이 먼저 필요 |
| 업종·테마 | **동종 기업 묶음(RAG)** + 대표 종목 주가 | 한 회사가 아니라 업종 전체를 봐야 함 |
| 시장 전체 | **거시지표(조회)** | 금리·환율·물가 흐름이 본질 |

### 5-4. 끌어온 자료는 출처로 보여준다 (P4)

가져온 1차 자료(공시 원문, 보고서 출처)는 버리지 않고 **콘텐츠에 출처 배지로 노출**한다. "리딩방·광고와 다르다"는 증거가 되기 때문이다. (이를 담을 출처 필드는 아직 없어 신설이 필요하다 → 개선과제)

## 6. 콘텐츠는 어떻게 구성되나

### 6-1. 공통 원자 = 이슈 도슨트

모든 콘텐츠는 하나의 **이슈 도슨트**에서 모양만 바꿔 재합성된다. 한 번 만든 도슨트를 재활용하므로 추가 비용이 낮다.

```
같은 이슈로 묶기(클러스터)
  → 분류 (scope · frame)
  → 추가자료 붙이기 (§5)
  → 해설 4단 (frame이 질문 4개를 고정, §3)
  → 연결 모듈 (관련 업종·기업)
  → 퀴즈
```

### 6-2. 콘텐츠 10종과 푸는 고민

| 콘텐츠 | 푸는 고민 | 푸는 방식 |
| --- | --- | --- |
| ① 이슈 도슨트 | P1 | 4단 해설 + 내 종목 앵커 |
| ② 클릭형 용어카드 | P1 심화 | 본문 속 "컨센서스" 탭 → 즉시 풀이 |
| ③ 전문가의 시각 | P1·P4 | "왜 그렇게 읽는지" 근거와 함께 |
| ④ 리스크 해설 + 손실 복기 | P2 | ★오해방지 단 + "왜 빠졌나·얼마나 걱정" |
| ⑤ 출처 배지·원문 링크 | P4 | §5에서 끌어온 1차 출처 노출 |
| ⑥ 익명 AI 1:1 질의 | P5 | 사람 없이 "기초도 물어봐도 돼요" |
| ⑦ 하루 5분 아침 브리핑 | P6 | 오늘의 3~5개를 한 줄씩 |
| ⑧ 왕초보 입문 트랙 | P3 | 0~3단계 단계별 가이드 |
| ⑨ 퀴즈 → 다음 이슈 루프 | 학습 | 이해 확인 + "읽는 법"이 남음 |
| ⑩ 단계 학습 커리큘럼 | P3 심화 | 용어 진도 + 오답 복습 + 읽는 유형별 모음 |

고민에서 출발해 끝까지 한 줄로 따라간다.

| 고민 | 주로 걸리는 유형·거리 | 붙이는 추가자료 | 산출 콘텐츠 | 푸는 방식 |

| **P1** 독해 | 실적·업황 / D칸 | 재무·사업보고서 | ①도슨트 ②용어카드 ⑨퀴즈 | 4단 해설로 떠먹이고 읽는 법을 남김 |

| **P3** 시작점 | 제도 / D칸 | 거시·업종매핑 | ⑧입문트랙 ⑩커리큘럼 | 단계별 코스 + D칸 읽기 연습 |
| **P4** 신뢰 | 새 계획·전문가 의견 / 전 거리 | 공시 원문(출처)·주가 | ⑤출처배지 ③전문가시각 | 1차 출처와 근거를 눈에 보이게 |
| **P5** 위축 | 전 유형 | — | ⑥익명 AI 질의 | 사람 없이, 무시 안 하는 톤 |
| **P6** 시간 | 제도 / A·B칸 | 내 종목 주가·공시 | ⑦아침 브리핑 | 오늘의 3~5개만 한 줄씩 |
