# 하루 세 가지 학습 MVP

## 목표

뉴스를 더 많이 보여주는 피드가 아니라, 하루 최대 세 이슈를 이해하고 끝낼 수 있는 학습
경험을 제공합니다.

| 순서 | 역할 | 선택 기준 |
|---:|---|---|
| 1 | 내 관심 | 선택한 회사·섹터와 직접 겹치는 최신 주요 이슈 |
| 2 | 시장 맥락 | 관심사와 연결되는 시장 전체 이슈, 없으면 최신 시장 이슈 |
| 3 | 시야 넓히기 | 관심사와 겹치지 않고 앞선 이슈와 섹터가 다른 주요 이슈 |

후보가 세 개보다 적으면 실제 생성된 수만 반환합니다. 중요하지 않은 콘텐츠로 빈자리를
억지로 채우지 않습니다.

## API

- `GET /api/v1/learning/today?sector_ids=...&company_ids=...`
  - 최근 7일의 발행 가능한 해설 중 stable cluster를 중복 제거합니다.
  - DB의 최신 실행일·중요도 순서를 유지하면서 세 역할을 채웁니다.
  - 각 이슈의 `kind=issue` 퀴즈 한 문제만 정답 없이 반환합니다.
- `POST /api/v1/learning/today/{issue_id}/quiz`
  - 선택지를 채점하고 정답, 해설을 반환합니다.
  - 게스트는 저장하지 않고, 로그인 사용자는 `user_issue_activities`에 완료를 저장합니다.

기사 상세 내용은 기존 `GET /api/v1/issues/{issue_id}`를 사용합니다. 관심사 태그는 삭제된
`issue_docent.*_ids`가 아니라 정규화된 단일 소스인 `news_analysis.company_ids`와
`news_analysis.sector_ids`를 사용합니다.

## 프론트 상태

게스트 우선 MVP라 관심사와 당일 계획, 완료 이슈 ID를 브라우저에 저장합니다. 당일 계획은
한국 시간 날짜와 관심사 조합이 같으면 재사용합니다. 로그인과 서버 동기화는 기존 인증 API를
활용하는 다음 단계로 둡니다.

## 완료 조건

각 이슈는 정답 여부와 무관하게 퀴즈를 한 번 제출하면 완료됩니다. 반환된 모든 이슈를
완료하면 오늘의 학습이 끝납니다.
